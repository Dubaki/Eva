---

## 📋 О продукте
**EVA** — Telegram Mini App (TMA), проводящее психологическую диагностику по 5 шкалам искажённых опор. Пользователь проходит тест, получает визуализированный результат (Доминирующая опора с ФОТО). Теневая (вторичная) опора открывается через реферальную воронку. Внедрена система возврата (Retention) через 60 дней, сохранение прогресса и продвинутая Панель Администратора для работы с "горячими" лидами.

---

## 🏗️ Архитектура (Обновлено)
### Стек
- **Framework:** Next.js 14 (App Router)
- **Database:** Supabase (PostgreSQL + RLS)
- **Background Jobs:** Supabase Edge Functions + pg_cron
- **Bot Logic:** Telegram Bot API (webhook) + Edge Functions

### Ключевая логика (Workflow)
1. **Сохранение результатов (СТРОГИЙ ПОРЯДОК):**
   - Приложение сначала обязано сделать `UPSERT` в таблицу `profiles` (чтобы избежать ошибки Foreign Key).
   - Только после этого делается `INSERT` в `test_results`.
2. **Система уведомлений (Edge Functions):**
   - `process-bot-notifications`: Срабатывает по вебхуку БД. Отправляет результат с ФОТО при новом `INSERT` в `test_results`. Отправляет смешанную опору (Теневую) при обновлении `invites_count` >= 2 в таблице `profiles`.
3. **Система возврата (Cooldown & Cron):**
   - **Frontend:** Блокировка кнопки "Пройти тест", если дата последней записи пользователя < 60 дней. Доступ к результатам и рефералке сохраняется.
   - **Backend:** `pg_cron` раз в сутки вызывает Edge Function `send-periodic-reminders`. Функция ищет юзеров, у которых прошло 60 дней И колонка `reminded_at IS NULL`.
4. **Виральная механика (Referrals):**
   - Используются **Direct Mini App Links** (`?startapp=TG_ID`).
   - Инкремент рефералов срабатывает строго после прохождения Subscription Gate (проверки подписки).

## 📐 База данных (Актуальная схема)
- **`profiles`:** - `tg_id` (Primary Key)
  - `username` (Отображается в CRM)
  - `invites_count` (Счетчик рефералов, обновляется при подписке)
  - `last_test_date` (Дата последнего теста для frontend-блокировки)
  - `reminded_at` (timestamptz, защита от спама Cron-напоминаний)
  - `current_step` (text/int, сохранение прогресса теста при выходе)
  - `contact_author_clicked` (boolean, индикатор "горячего" лида для CRM)
- **`test_results`:** - Foreign Key привязан к `profiles.tg_id`.
  - Хранит `primary_support` (одна буква) и `secondary_support` (одна буква).
