<context>
Приложение готово к продакшену. Чтобы реферальная система работала внутри Telegram, нам нужен Webhook, который будет обрабатывать команды `/start` (в том числе с реферальными параметрами) и отправлять пользователю кнопку для открытия TMA (Telegram Mini App).
</context>

<task>
1. Создать API Route `app/api/webhook/telegram/route.ts` (метод POST).
2. Реализовать логику:
   - Принять JSON от Telegram (объект Update).
   - Если текст начинается с `/start`, извлечь реферальный код (если есть).
   - Отправить пользователю приветственное сообщение (через `fetch` к `api.telegram.org/bot<TOKEN>/sendMessage`).
   - К сообщению прикрепить Inline-кнопку (тип `web_app`), которая открывает наше Mini App.
3. Проверить безопасность: убедиться, что эндпоинт не упадет, если придет неожиданный формат данных.
4. Вывести в консоль (или лог) четкую curl-команду для установки вебхука (`setWebhook`), чтобы Заказчик мог выполнить её после деплоя.
5. Перенести этот файл в tasks/done/ после завершения.
</task>

<rules>
- НЕ использовать сторонние библиотеки вроде `telegraf` или `node-telegram-bot-api` — они избыточны. Использовать только нативный `fetch` (REST API).
- Использовать `process.env.BOT_TOKEN`.
- Исполнитель: Claude Code.
</rules>
