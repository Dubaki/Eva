<context>
При входе в бота (команда /start) пользователь не видит приветственного сообщения и картинки. Нужно реализовать строгую воронку подписки на канал до входа в Mini App, согласно ТЗ.
</context>

<task>
1. В обработчике вебхуков Telegram (например, `app/api/webhook/telegram/route.ts` или где настроена маршрутизация) найти или создать обработку команды `/start`.
2. При получении `/start` бот должен отправлять фото (любое из папки `public`) и текст: 
   "Привет! Перед тем как начать, подпишись на канал. Там я даю информацию, которую не даю больше нигде."
3. К сообщению прикрепить инлайн-клавиатуру с двумя кнопками:
   - "Подписаться" (кнопка-ссылка (url) на канал Заказчика).
   - "Я подписалась" (callback-кнопка для проверки).
4. Реализовать обработку callback-запроса от кнопки "Я подписалась":
   - Бот должен вызвать метод `getChatMember`, чтобы проверить статус пользователя в канале.
   - Если статус "member", "administrator" или "creator" -> отправить сообщение об успехе с кнопкой Web App для открытия приложения.
   - Если статус другой -> показать уведомление (answerCallbackQuery), что подписка не найдена.
5. Перенести этот файл из tasks/todo/ в tasks/done/ после завершения.
</task>

<rules>
- Исполнитель: Qwen Code (задача на логику бэкенда и API Telegram).
- ID канала для проверки подписки и ссылку на канал вынести в переменные окружения (`TELEGRAM_CHANNEL_ID`, `TELEGRAM_CHANNEL_URL`).
- Не пускать пользователя в приложение до успешного прохождения проверки.
</rules>
