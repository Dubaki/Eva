<context>
Текущая проблема: Сообщения от бота не доходят до пользователей через Vercel. Доступ в Админ-панель заблокирован из-за ошибок типизации ID.
Решение:
1. Перенос логики бота в Supabase Edge Functions (автоматическая отправка при изменении в БД).
2. Доступ в Админку через скрытый PIN-код.
3. Система управления подарками через БД и Админ-панель.
</context>

<task>
**ЧАСТЬ 1: База данных (Supabase)**
1. Создать таблицу `app_settings` (columns: `key` text primary key, `value` text).
   - Вставить дефолтные ссылки для подарков: `gift_money`, `gift_relations`, `gift_health`, `gift_other`.
2. В таблице `profiles` убедиться, что есть поля: `selected_sphere` (для подарка), `dominant_trait`, `shadow_trait`, `referrals_count`.

**ЧАСТЬ 2: Supabase Edge Function (Логика Бота)**
3. Создать Edge Function `process-bot-notifications`:
   - Триггер: `ON INSERT OR UPDATE` в таблице `profiles`.
   - Логика:
     А) Если `dominant_trait` только что записан (был null, стал не null): Отправить пользователю (tg_id) сообщение с Картинкой (абсолютный URL) и Текстом доминирующей опоры.
     Б) Если `referrals_count` изменился и стал равен 2: Найти в словаре (маппинг из Task 068) текст Смешанной опоры и отправить пользователю в чат.
   - Использовать `Deno.env.get("TELEGRAM_BOT_TOKEN")` для запросов к Telegram API.

**ЧАСТЬ 3: Админ-панель и Секретный вход**
4. Внедрить **Secret PIN Access**:
   - На главной странице добавить скрытый триггер: 7 быстрых кликов по заголовку "У каждого человека есть внутренняя «опора»".
   - По клику открывается `window.prompt("Введите PIN-код")`.
   - Если код `2026` (вынести в константу), сохранить в `localStorage.setItem('isAdmin', 'true')` и перенаправить в `/admin`.
   - В самой Админке проверять этот флаг в дополнении к проверке списка ID.
5. Создать вкладку **"🎁 Подарки"** в Админ-панели:
   - Выводить 4 поля ввода для ссылок на подарки (Деньги, Отношения, Здоровье, Другое).
   - Реализовать сохранение этих ссылок в таблицу `app_settings`.

**ЧАСТЬ 4: Фронтенд - Выдача подарка и UX**
6. Обновить логику кнопки `[Забрать подарок]`:
   - При нажатии приложение должно:
     1. Запросить из таблицы `app_settings` ссылку, соответствующую `selected_sphere` пользователя.
     2. Открыть эту ссылку через `window.Telegram.WebApp.openLink(url)`.
     3. Выполнить `window.Telegram.WebApp.close()` через 500мс.
7. Обновить логику выхода для кнопок "Пробой" и "Пирамида":
   - Сначала `openTelegramLink(...)`, затем `close()` через 500мс.

**ЧАСТЬ 5: Подготовка "Машины времени"**
8. Добавить в таблицу `profiles` колонку `reminded_at` (timestamp, nullable). Мы будем использовать её позже для Cron-задачи на 60 дней.
