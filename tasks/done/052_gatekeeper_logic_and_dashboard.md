Необходимо реализовать жесткую проверку прав доступа (подписка и кулдаун) и создать постоянный экран "Дашборд результатов" для тех, кто уже прошел тест.
</context>

<task>
**ЧАСТЬ 1: Глобальная проверка (Frontend Entry Point)**
1. В главном файле инициализации приложения (например, `App.tsx` или `layout.tsx`) добавить проверку статуса пользователя при загрузке:
   - Если `user.is_subscribed === false` -> Блокировать весь интерфейс. Показать экран-заглушку: "⚠️ Доступ закрыт. Пожалуйста, подпишись на канал в боте и нажми 'Я подписалась'".
   - Если `user.last_test_date` был менее 60 дней назад -> Автоматически перенаправлять пользователя с экрана "Начать тест" на экран "Результаты".

**ЧАСТЬ 2: Логика Кулдауна (Backend)**
2. При завершении теста (сохранение результатов в Supabase) обязательно записывать текущую дату в колонку `last_test_date`.
3. Реализовать серверную проверку: если пользователь пытается отправить ответы на тест повторно раньше чем через 2 месяца, база данных должна отклонять запрос.

**ЧАСТЬ 3: Дашборд результатов (Экран для возвращающихся)**
4. Если тест пройден, пользователь всегда попадает на этот экран:
   - **Сверху:** Доминирующая опора (Картинка и описание, которые уже сделали).
   - **Ниже:** Блок "Теневая опора" (2-е место).
     - Если `referrals_count < 2`: Показать "замок" 🔒, прогресс-бар (0/2 или 1/2) и кнопку "Пригласить подруг".
     - Если `referrals_count >= 2`: Разблокировать и показать вторую картинку (pleaser, hero и т.д. согласно 2-му месту) и текст.
   - **Кнопка Этапа 5:** Если пользователь еще не прошел 3 вопроса квалификации, показать кнопку "Разобраться с этим ➔". Если прошел — показать "Мой план работы".

**ЧАСТЬ 4: Синхронизация рефералов**
5. Убедиться, что при открытии приложения оно запрашивает актуальный `referrals_count` из Supabase, чтобы пользователь видел изменения в реальном времени, когда друзья подключаются.

6. Сделать git push и перенести в done/.
</task>
