<task>
**ЧАСТЬ 1: Конфигурация VIP-пользователей**
1. В основном файле логики доступа (Mini App) и в API-роутах (Backend) создать константу с массивом ID тестировщиков:
   `const TESTER_IDS = [1149371967, 5930269100, 1419397753];`

**ЧАСТЬ 2: Обход блокировок (Gatekeeper Override)**
2. В логике Frontend (где проверяются права доступа на старт теста):
   - Если `TESTER_IDS.includes(user.tg_id)`:
     - Всегда считать пользователя подписанным (пропускать заглушку "Доступ закрыт").
     - Игнорировать проверку `last_test_date` (всегда показывать кнопку "Пройти тест", не форсировать редирект на дашборд результатов).
3. В логике Backend (сохранение результатов):
   - Если `TESTER_IDS.includes(user.tg_id)`, разрешать перезапись результатов теста, даже если прошло меньше 2 месяцев.

**ЧАСТЬ 3: Связь с Автором (Этап 6)**
4. На финальном экране (Этап 6), где расположены кнопки "Быстрый формат" и "Мягкий формат", подготовить правильные ссылки:
   - Создать константу: `const AUTHOR_USERNAME = "evapatrakhina";`
   - Использовать её в кнопках через API Telegram: 
     - Кнопка 1: `onClick={() => window.Telegram.WebApp.openTelegramLink('https://t.me/' + AUTHOR_USERNAME + '?text=Пробой')}`
     - Кнопка 2: `onClick={() => window.Telegram.WebApp.openTelegramLink('https://t.me/' + AUTHOR_USERNAME + '?text=Пирамида')}`

5. Перенести в done/.
</task>
