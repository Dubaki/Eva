<context>
Нужно реализовать автоматическое напоминание пользователю о том, что прошло 2 месяца с момента последнего теста и пора обновить данные об опорах.
</context>

<task>
1. **Настройка Edge Function (Cron):**
   - Создать новую Edge Function `send-periodic-reminders` или добавить логику в существующий планировщик.
   - Скрипт должен запускаться раз в сутки.
   - Логика: Выбрать всех пользователей из `test_results`, у которых дата `created_at` последней записи (самой свежей для этого `tg_id`) равна ровно 60 дням от текущей даты.

2. **Сообщение в Telegram:**
   - Если условие (60 дней) выполнено, отправить пользователю сообщение:
     "Привет! Прошло уже 2 месяца с твоего последнего теста. Твои опоры могли трансформироваться или укрепиться. Давай проверим твое актуальное состояние?"
   - Добавить под текстом Inline-кнопку: "Пройти тест заново", ведущую в Mini App.

3. **Блокировка кнопки в Mini App:**
   - На фронтенде Mini App реализовать проверку: если у пользователя есть запись в `test_results`, созданная менее 60 дней назад, кнопка "Пройти тест" должна быть неактивна (или заменена на таймер обратного отсчета).
   - Текст при попытке нажать: "Опора еще формируется. Повторный тест будет доступен через [X] дней".

4. **Интеграция с Supabase Cron:**
   - Включить расширение `pg_cron` в Supabase (если еще не включено).
   - Настроить расписание вызова функции через `http` запрос.
</task>
