<context>
Требуется расширение функционала CRM (пункты 8 и 9). Нужно отслеживать "горячих" лидов, которые нажали кнопку "Связь с Автором", и иметь возможность отправлять рассылки не всем подряд, а выбранным пользователям из таблицы.

<task>
1. Добавить в таблицу `profiles` в Supabase колонку `contact_author_clicked` (boolean, default: false).
2. На фронтенде (в TMA) привязать к кнопке "Связь с Автором" вызов API, который переводит `contact_author_clicked` в `true` для данного `tg_id`.
3. В Админ-панели добавить в таблицу индикатор/колонку (например, бейдж или иконку), показывающий статус "Запросил связь" (`contact_author_clicked == true`).
4. В Админ-панели добавить возможность выбора пользователей (чекбоксы в таблице) для массовой рассылки.
5. Изменить логику модального окна Broadcast: если выбраны пользователи, отправлять сообщения только массиву их `tg_id`, иначе — предлагать отправку "Всем".
6. Перенести этот файл из tasks/todo/ в tasks/done/ после завершения.
</task>
