<context>
Telegram блокирует автоматическую отправку сообщений от лица пользователя. Чтобы не терять лидов на Этапе 6, нам нужна система "Теневого пинга": при клике на оффер бот должен отправлять Автору уведомление с контактами лида.
</context>

<task>
**ЧАСТЬ 1: Backend (Новый API Route для уведомлений)**
1. Создать новый обработчик: `app/api/notify-author/route.ts` (или аналогичный).
2. Логика API:
   - Принимает POST-запрос с данными: `userId`, `firstName`, `username`, `selectedFormat` ("Пробой" или "Пирамида").
   - Использует `bot.sendMessage` (из Telegram API) для отправки сообщения на ID Автора (`1149371967`).
   - Текст сообщения (в HTML):
     🔥 <b>Горячий лид!</b>
     Пользователь только что выбрал формат: <b>{selectedFormat}</b>
     
     👤 Имя: {firstName}
     🔗 Профиль: {username ? `@${username}` : `<a href="tg://user?id=${userId}">Ссылка на аккаунт</a>`}
     
     <i>Если он не написал сам в течение пары минут — напиши ему первая!</i>

**ЧАСТЬ 2: Frontend (Вызов API при клике)**
3. На Этапе 6 в компоненте с кнопками оффера ("Быстрый формат", "Мягкий формат"):
   - Изменить обработчик `onClick`.
   - Теперь при клике нужно СНАЧАЛА сделать асинхронный `fetch` запрос на наш новый роут `/api/notify-author` (передать данные пользователя и выбранный формат).
   - СРАЗУ ПОСЛЕ (не дожидаясь ответа, чтобы не было задержки для пользователя) вызвать `window.Telegram.WebApp.openTelegramLink(...)`.

4. Сделать git push и перенести в done/.
</task>
